## Overview

This repository uses **deploy-managed Copilot Chat Agents** to accelerate platform deployment and operations. These are conversational agents in your IDE that help you configure infrastructure, validate security, integrate source platforms, and deploy Open Horizons.

---

## Copilot Chat Agents

These agents live in GitHub Copilot Chat. The `@deploy` agent is the primary entry point and hands off to specialist agents when needed.

| Agent | Specialized Role | When to use? |
| :--- | :--- | :--- |
| `@deploy` | Deployment Orchestrator | "Deploy the platform to my AKS cluster." |
| `@terraform` | IaC & Azure | "Help me configure the AKS module." |
| `@security` | Compliance | "Am I following the security baseline?" |
| `@sre` | Observability | "How do I debug this production error?" |
| `@backstage-expert` | Backstage Portal | "Configure GitHub auth and register Golden Paths." |
| `@azure-portal-deploy` | Azure Portal Infrastructure | "Provision AKS, Key Vault, PostgreSQL, and ACR." |
| `@github-integration` | GitHub Integration | "Configure a GitHub App and org discovery." |
| `@ado-integration` | Azure DevOps Integration | "Configure Azure DevOps repository discovery." |
| `@hybrid-scenarios` | GitHub + ADO Coexistence | "Choose the right hybrid scenario for this client." |

### How to use

In VS Code or GitHub.com, type `@agent-name` in the Copilot Chat window.

> **Example:** `@deploy help me check if I have all prerequisites installed.`

## Workflow Patterns

### The "Fork-to-Deploy" Journey

1. **Start:** User forks repo.
2. **Guide:** User asks `@deploy`: "Help me get started".
3. **Config:** `@deploy` helps user create `.tfvars`.
4. **Deploy:** User runs `./scripts/platform-bootstrap.sh --environment dev`.
5. **Action:** Terraform provisions Azure infrastructure.
6. **Result:** Foundation layer is live.

### Platform Deployment Flow

1. `@deploy` - Orchestrate the end-to-end deployment.
2. `@terraform` - Validate or author Terraform module changes.
3. `@azure-portal-deploy` - Provision Azure portal infrastructure when needed.
4. `@backstage-expert` - Configure the Backstage portal.
5. `@github-integration`, `@ado-integration`, or `@hybrid-scenarios` - Configure source-control integration.
6. `@security` - Review identity, RBAC, secrets, and network posture.
7. `@sre` - Verify health and observability after deployment.

## Best Practices

1. **Start with `@deploy`:** Use the deployment orchestrator as the default entry point for platform work.
2. **Use Handoffs:** Agents are aware of each other. If `@deploy` sees you need Terraform, Backstage, identity, or post-deploy verification help, it will route to the right specialist.
3. **Trust but Verify:** Always review the code generated by Copilot agents before committing.
4. **Security First:** Never put real secrets in chat. Use Key Vault references.
